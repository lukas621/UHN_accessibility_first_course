# Illustration Style System — UHN Accessibility First Course Series

## Purpose

Define the visual illustration approach for the 18-course Accessibility First series. This document guides all illustration work — whether produced by a human illustrator, AI generation tools, or a combination — to ensure consistency, inclusivity, and brand alignment across all courses.

---

## Design Principles

### 1. Inclusive Representation

- Depict people across a full range of disabilities, ages, races, ethnicities, genders, and body types
- Show people WITH disabilities, not people DEFINED by disabilities
- Avoid pity framing, inspiration framing, or medical-model imagery
- Show people in active, professional, everyday contexts — not passive or isolated
- Include Indigenous peoples with cultural accuracy and respect

### 2. Warm Professionalism

- Healthcare setting but not clinical or sterile
- Approachable, not childish
- Serious enough for professional development, human enough for empathy-based learning
- Consistent with UHN's institutional brand voice

### 3. Accessibility in the Art Itself

- Sufficient contrast for low-vision learners
- Avoid relying on color alone to convey meaning
- Clear, uncluttered compositions
- Every illustration must have meaningful alt text written at the storyboard stage
- Avoid flashing, strobing, or high-motion elements

### 4. Consistency Across 18 Courses

- Same art style, color palette, line weight, and proportions across all guides
- Recurring characters or character archetypes to build familiarity
- Shared visual language for frameworks (Decision Path, four-quadrant model, guiding principles)

---

## Recommended Illustration Style

### Primary Style: Flat Vector with Soft Gradients

**Why this style:**
- Scales well across screen, PPT, Word export, and Storyline
- Reproduces cleanly at any size
- Easy to maintain consistency across 18 courses
- Can be produced by illustrators or AI tools
- Accessible: clean lines, strong contrast, no visual noise

**Characteristics:**
- Clean outlines (2-3px weight)
- Flat color fills with subtle gradients for depth
- Rounded, friendly shapes
- Minimal shadows (soft drop shadow only where needed)
- No photorealistic textures
- Simple, expressive faces (enough detail for emotion, not photorealism)
- Consistent head-to-body proportions across all characters

### Color Palette

Derive from UHN brand templates in `02-branding-and-style/`. Layer on:

| Role | Usage |
|---|---|
| Primary brand color | Headers, key UI elements, character accents |
| Secondary brand color | Supporting elements, backgrounds |
| Neutral warm | Skin tones (diverse range), backgrounds |
| Neutral cool | Healthcare environments, equipment |
| Accessibility accent | Highlights for accessibility-related callouts |
| Alert / caution | SME review flags, high-risk content markers |

### Skin Tone Palette

- Minimum 6 distinct skin tones used consistently
- Based on inclusive illustration standards (e.g., Skin Tone Guide by Ange Albertini or similar)
- Never use a single "default" skin tone

## Illustration Categories

### Category 1: Character Illustrations

**Purpose:** Represent learners, healthcare workers, patients, support persons, and community members in scenarios.

**Requirements per course:**
- 2-4 primary character illustrations
- Characters should reflect the disability focus of each guide
- Show characters in context: hospital, clinic, community, workplace
- Include assistive devices naturally (wheelchairs, white canes, hearing aids, service animals, communication boards, etc.)

**Sensitivity rules:**
- Never depict disability as the sole defining trait
- Show agency — characters making decisions, communicating, participating
- Avoid "before/after" or "problem/solution" framings
- Guide 18 (Indigenous Peoples and Accessibility): consult Indigenous advisors for character representation

**Character archetypes to develop:**
1. Healthcare worker (nurse, doctor, allied health, admin — varied roles)
2. Patient/client (diverse disabilities, ages, backgrounds)
3. Support person (family member, PSW, interpreter, guide)
4. Colleague/peer (workplace context)
5. Community member (public-facing scenarios)

### Category 2: Framework & Concept Illustrations

**Purpose:** Visualize the shared instructional frameworks that repeat across all 18 courses.

| Framework | Visual Treatment | Reuse |
|---|---|---|
| Accessibility Decision Path (Pause → Listen → Apply → Adapt → Seek Support) | Horizontal step diagram with icons, each step has a distinct icon | All 18 courses |
| Accessibility in Practice (Awareness, Communication, Environment, Response) | Four-quadrant diagram with icons | All 18 courses |
| Guiding Principles (dignity, independence, inclusion, proactive barrier removal, equity) | Icon cards or principle badges | All 18 courses |
| MAP Action Planning | Template/worksheet visual | All 18 courses |

**These must be designed ONCE and reused consistently.** Variations per course can highlight the relevant quadrant or step, but the base visual stays identical.

### Category 3: Scenario Illustrations

**Purpose:** Support the scenario-based learning screens. Each course has 2-4 scenarios.

**Estimated volume:** 18 courses × 3 scenarios average = ~54 scenario illustrations

**Requirements:**
- Show the scenario setup (who, where, what's happening)
- Neutral enough to support multiple decision branches
- Do NOT show the "answer" — the learner decides
- Include environmental context (hospital room, reception desk, community space, etc.)

**Approach options:**
- **Option A: Illustrated scenes** — custom illustration per scenario (highest quality, highest cost)
- **Option B: Character + background compositing** — reusable character assets placed on reusable backgrounds (medium cost, good consistency)
- **Option C: AI-generated with human review** — generate scenario illustrations using AI tools, reviewed for accuracy and sensitivity (lowest cost, requires strong review process)

**Recommended: Option B** — Build a character and background asset library, compose scenes. Supplement with AI generation for backgrounds/environments.

### Category 4: Icons & Micro-illustrations

**Purpose:** Small visual elements for navigation, interaction types, and content markers.

**Needed:**
- Interaction type icons: click-reveal, scenario, decision branch, knowledge check, reflection, video, job aid
- Content marker icons: tip, warning, reflection prompt, key concept, SME flag
- Navigation icons: next, back, menu, resources, glossary
- Disability-specific icons (per guide): vision, hearing, physical, mental health, intellectual, non-visible, aging, neurodiversity, trauma, crisis, service animals, support persons, assistive devices, communication
- Shared framework icons (Decision Path steps, four-quadrant areas, guiding principles)

**Estimated volume:** ~80-100 icons total

**Style:** Match the flat vector style. 2-color maximum per icon. Must be legible at 24px and 48px.

### Category 5: Infographics & Diagrams

**Purpose:** Explain processes, data, or relationships visually.

**Per course (estimated):**
- 1 concept map or relationship diagram
- 1 process/workflow visual
- 1 data or statistics visual (if applicable)

**Estimated volume:** 18 courses × 2 average = ~36 infographics

## AI-Assisted Illustration Workflow

For courses about accessibility, AI-generated images require extra scrutiny. Recommended workflow:

### Generation

1. Write a detailed prompt including: style reference, scene description, character details, disability representation notes, what NOT to include
2. Generate 3-5 variants
3. Human review for:
   - Accurate disability representation (e.g., correct wheelchair type, realistic assistive device)
   - Inclusive and respectful framing
   - Cultural accuracy (especially Guide 18)
   - Consistency with established style
   - UHN brand alignment
4. Select and refine (manual touch-up in Illustrator/Figma if needed)
5. Write alt text

### Tools for AI Generation

| Tool | Best For | Limitations |
|---|---|---|
| Midjourney | High-quality character and scene illustrations | Style consistency requires careful prompting; limited control over details |
| DALL-E 3 | Quick concept exploration, backgrounds | Can struggle with consistent character representation |
| Adobe Firefly | Brand-safe generation, integrates with Illustrator | Newer, evolving capabilities |
| Stable Diffusion (with ControlNet) | Highest control over poses and compositions | Requires more technical setup |
| Canva AI | Quick infographics and simple diagrams | Limited for custom character work |

### Human Review Checklist for AI-Generated Images

- [ ] Does the image avoid stereotypical or pitying disability representation?
- [ ] Are assistive devices depicted accurately?
- [ ] Is the person shown with agency and dignity?
- [ ] Does skin tone representation match the intended diversity?
- [ ] Is the healthcare setting realistic?
- [ ] Does the style match established illustration guidelines?
- [ ] Is the image free of AI artifacts (extra fingers, distorted text, impossible anatomy)?
- [ ] Has alt text been written?
- [ ] For Indigenous content: has the image been reviewed by an Indigenous advisor?
