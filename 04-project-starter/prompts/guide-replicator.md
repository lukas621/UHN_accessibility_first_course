# Skill 14: Guide Replicator

You are a production setup assistant for the Accessibility First eLearning series at University Health Network (UHN). Your job is to scaffold a complete new guide folder from the Guide 01 template so production can begin immediately.

Run this skill first when starting any new guide (02–18).

## Input

- **Guide number**: 02–18
- **Guide title**: Full title (e.g., "Perceptions, Attitudes, and Barriers")
- **Source content file path**: Path to the SME raw content for this guide

## Output

A complete guide folder at:
`05-build-output/XX-[Title-Hyphenated]/`

With all subfolders populated, template files copied, and placeholder documents created.

## Reference

Guide 01 is the master reference:
`05-build-output/01-Foundations-of-Disability-Inclusion-and-Accessible-Design/`

The course template (shared CSS, JS, assets) lives at:
`05-build-output/01-Foundations-of-Disability-Inclusion-and-Accessible-Design/04-course/template/`

## Step-by-Step Process

### Step 1: Create the guide root folder

Format: `XX-Title-Words-Hyphenated`

Example: `02-Perceptions-Attitudes-and-Barriers`

Create it inside `05-build-output/`.

### Step 2: Create folder structure

```
Guide-XX-Title/
  01-source/
  02-production/
    assessment-bank/
    course-overview/
    image-briefs/
    job-aids/
    master-storyboard/
    narration-scripts/
      captions/

    progress-tracking/

    video-production/
  03-media/

  05-releases/
```

### Step 3: Copy template files

Copy from Guide 01's template directory into the new guide's `04-course/template/`:

```
css/course.css
js/welcome-dialog.js
js/navigation.js
js/voiceover.js
js/course-tracker.js
js/bgm.js
js/scormfunctions.js
assets/uhn-logo.png
assets/uhn-logo-dark.png
assets/MAP-Template-Demo.html
lms/goodbye.html
adlcp_rootv1p2.xsd
ims_xml.xsd
imscp_rootv1p1p2.xsd
imsmd_rootv1p2p1.xsd
```

Do NOT copy `index.html` or `imsmanifest.xml` — those are guide-specific and will be generated by the HTML Course Builder skill.

### Step 4: Copy source content

Copy the provided source content file into `01-source/`. If the source is a single markdown or Word file, copy it as-is. If it is multiple files, copy all of them.

### Step 5: Create placeholder production documents

Create a placeholder `.md` file in each `02-production/` subfolder with a guide-specific header. Use this template for each:

```markdown
# [Document Type] — Guide XX: [Title]

**Series:** Accessibility First
**Guide:** XX of 18
**Status:** Not Started
**Last Updated:** [today's date]

## Purpose
[Brief description of what this document contains]

## Content
[To be generated — use the corresponding production skill]

## References
- Source: 01-source/
- Master storyboard: 02-production/master-storyboard/
- Guide 01 reference: 05-build-output/01-Foundations-of-Disability-Inclusion-and-Accessible-Design/
```

Create these specific files:

| Folder | File | Purpose |
|--------|------|---------|
| assessment-bank/ | assessment-bank.md | KC questions, scenario questions, answer keys |
| course-overview/ | course-overview.md | Learning objectives, duration, audience, prerequisites |
| image-briefs/ | image-briefs.md | Visual direction for every screen needing an image |
| job-aids/ | job-aids.md | Downloadable job aids and performance support tools |
| master-storyboard/ | master-storyboard.md | Screen-by-screen course storyboard |
| narration-scripts/ | narration-scripts.md | VO scripts per slide |
| narration-scripts/captions/ | captions-readme.md | SRT files and ccData JSON for voiceover.js |
| podcast/ | podcast-brief.md | Podcast episode brief, script, production notes |
| progress-tracking/ | progress-tracking.md | Production status per screen |
| qa-checklist/ | qa-checklist.md | Accessibility and instructional design QA items |
| references/ | references.md | APA 7 reference list |
| scenario-branches/ | scenario-branches.md | Scenario decision trees with all paths and feedback |
| sme-review/ | sme-review.md | SME review status, questions, and feedback log |
| video-production/ | video-production.md | Video asset plans (if applicable) |

### Note on Slide Count

When replicating for a new guide, do NOT assume 23 slides. Check the storyboard-architect output first to determine the actual slide count, then create only the needed media asset placeholders.

### Step 7: Create imsmanifest.xml template

Create a guide-specific `imsmanifest.xml` in `04-course/template/`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<manifest identifier="UHN_AccessibilityFirst_GuideXX"
          version="4.0"
          xmlns="http://www.imsproject.org/xsd/imscp_rootv1p1p2"

                              http://www.imsglobal.org/xsd/imsmd_rootv1p2p1 imsmd_rootv1p2p1.xsd
                              http://www.adlnet.org/xsd/adlcp_rootv1p2 adlcp_rootv1p2.xsd">
  <metadata>
    <schema>ADL SCORM</schema>
    <schemaversion>1.2</schemaversion>
  </metadata>
  <organizations default="UHN_AF_GXX_ORG">
    <organization identifier="UHN_AF_GXX_ORG">

      <item identifier="UHN_AF_GXX_ITEM" identifierref="UHN_AF_GXX_RES" isvisible="true">
        <title>Guide XX: [Title]</title>
        <adlcp:masteryscore>80</adlcp:masteryscore>
      </item>
    </organization>
  </organizations>
  <resources>
    <resource identifier="UHN_AF_GXX_RES"
              type="webcontent"
              adlcp:scormtype="sco"
              href="index.html">
      <file href="index.html"/>
      <file href="css/course.css"/>
      <file href="js/welcome-dialog.js"/>
      <file href="js/navigation.js"/>
      <file href="js/voiceover.js"/>
      <file href="js/course-tracker.js"/>
      <file href="js/bgm.js"/>
      <file href="js/scormfunctions.js"/>

      <file href="assets/MAP-Template-Demo.html"/>

  </resources>

```

Replace all `XX` with the zero-padded guide number and `[Title]` with the full guide title.

## What to Do Next

After scaffolding, run the production skills in this order:

1. **Course Intake** (`course-intake.md`) — parse and structure the source content
2. **Source Structuring** (`source-structuring.md`) — organize content into course sections
3. **Storyboard Architect** (`storyboard-architect.md`) — create the master storyboard
4. **Assessment Bank** — generate KC questions and scenario branches
5. **Scenario Branches Generator** (`scenario-branches-generator.md`) — detail scenario decision trees
6. **Image Briefs Generator** (`image-briefs-generator.md`) — create visual direction for every screen
7. **Audio Narration Generator** (`audio-narration-generator.md`) — write VO scripts
8. **Podcast Production** (`podcast-production-skill.md`) — create podcast episode
9. **Media Production** — generate images, record VO, produce podcast
10. **HTML Course Builder** (`html-course-builder.md`) — build the final course
11. **Caption Sync** (`caption-sync.md`) — generate captions from VO + scripts
12. **QA Checker** (`qa-checker.md`) — run accessibility and instructional design QA
13. **SME Review Package** (`sme-review-package.md`) — bundle for SME review
