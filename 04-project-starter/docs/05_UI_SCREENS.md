# 05 UI Screens

## 1. Dashboard

Purpose: show the overall production health of all course projects.

Widgets:

- Projects in progress
- Courses by status
- Awaiting SME review
- Overdue tasks
- QA issues
- Exports ready
- Recent activity

Main table:

- Course title
- Guide number
- Stage
- Progress
- SME status
- QA status
- Due date
- Risk

## 2. Project / Course Series Page

Purpose: manage a multi-course series.

Sections:

- project summary
- series description
- shared framework
- target audience
- output requirements
- timeline
- list of course guides

Actions:

- add course
- upload source
- generate all blueprints
- generate status report
- export tracker

## 3. Source Material Hub

Purpose: manage uploaded SME/source materials.

Features:

- upload document
- paste text
- document parse status
- extracted sections
- source-to-course mapping
- content gap detection
- duplicate content detection

AI outputs:

- document summary
- key themes
- potential course/guide list
- SME clarification questions

## 4. Course Guide Detail Page

Purpose: view and manage one course/guide.

Tabs:

- Overview
- Source Sections
- Blueprint
- Storyboard
- SME Review
- Assessments
- Media Tasks
- QA
- Exports

## 5. Blueprint Page

Purpose: define the instructional architecture.

Fields:

- course goal
- audience summary
- learning objectives
- sub-learning outcomes
- module structure
- assessment strategy
- interaction strategy
- performance support strategy
- estimated seat time

Actions:

- generate blueprint
- regenerate selected section
- approve blueprint
- export blueprint

## 6. Storyboard Builder

Purpose: core production workspace.

Layout:

- table/grid view
- screen detail side panel
- source excerpt panel
- AI assist actions

Columns:

- screen #
- module
- title
- screen purpose
- on-screen text
- narration
- interaction
- visual/media
- assessment/reflection
- performance support
- SME review level
- accessibility note
- status

Actions:

- add screen
- generate screens from blueprint
- improve selected screen
- generate assessment
- generate visual prompt
- send to SME review
- export storyboard

## 7. SME Review Page

Purpose: prepare targeted SME review package.

Features:

- review questions by screen
- high-risk content list
- approval checklist
- unresolved issues
- change summary
- comments

SME review statuses:

- not sent
- sent
- in review
- changes requested
- approved

## 8. Assessment Page

Purpose: manage quiz and reflection items.

Fields:

- question type
- question stem
- options
- correct answer
- feedback
- Bloom's level
- Kirkpatrick level
- linked objective
- linked screen
- SME approval status

## 9. Media Production Page

Purpose: plan multimedia assets.

Media task types:

- video
- podcast
- voice-over
- infographic
- illustration
- scenario video
- HTML5 interaction
- job aid

Fields:

- title
- linked storyboard screen
- tool recommendation
- script/prompt
- production notes
- status

## 10. QA Page

Purpose: run and track QA issues.

Categories:

- accessibility
- instructional design
- content quality
- objective alignment
- SME approval
- media readiness
- handoff readiness

Actions:

- run QA
- mark resolved
- assign issue
- export QA report

## 11. Export and Handoff Page

Purpose: package deliverables.

Export options:

- Word storyboard
- Excel tracker
- PPT draft
- SME review package
- QA checklist
- media task list
- handoff checklist

Handoff sections:

- source files
- editable learning materials
- media files
- captions/transcripts
- Storyline/Rise/HTML5 files
- SCORM package
- LMS testing notes
- SME approval record
- evaluation tools
