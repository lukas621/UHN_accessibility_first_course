# 09 Accessibility First Pilot Use Case

## Purpose

This document defines how to use the Accessibility First course series as the first internal pilot for Course Factory OS.

## Source Material Pattern

The source material is a large accessibility-related course series with 17 guides. Each guide generally includes:

- guide title
- purpose
- Indigenous Peoples and Accessibility section
- how to use this guide
- Accessibility Decision Path
- primary focus
- learning emphasis
- core objectives
- Accessibility in Practice
- key themes
- guiding principles
- sections
- scenarios
- reflection questions
- inclusive practice tips
- applying insights
- My Action Planning / MAP
- references

## Shared Frameworks to Extract

### Accessibility Decision Path

Use this as a reusable interaction pattern:

```txt
Pause and Assess
→ Listen and Ask
→ Apply
→ Adapt
→ Seek Support
```

Potential eLearning treatment:

- horizontal step interaction
- branching scenario decision model
- downloadable quick-reference job aid
- repeated framework activity across all courses

### Accessibility in Practice Model

Use four interconnected areas:

```txt
Awareness
Communication
Environment
Response
```

Potential eLearning treatment:

- four-quadrant interaction
- scenario tagging activity
- reflection prompt

### Guiding Principles

Common principles:

- dignity
- independence/autonomy
- inclusion/participation
- proactive barrier prevention and removal
- equity, rights, and intersectionality

Potential eLearning treatment:

- principle cards
- matching activity
- scenario debrief framework

```txt

5. Scenario Practice
6. Knowledge Check

```

## Storyboard Pattern

Each course should generate 10-18 screens depending on complexity.

Recommended screen types:

- intro screen
- learning objectives
- context / why it matters
- core concept screen
- decision path interaction
- scenario screen
- reflection screen
- knowledge check
- practice tip reveal
- action planning screen
- resources / completion screen

## SME Review Rules

### High SME Review

- legal/compliance statements
- policy interpretation
- service animal access rules
- clinical safety or IPAC considerations
- Indigenous cultural safety content
- accessibility legislation interpretation

### Medium SME Review

- scenarios
- practice tips
- terminology
- assessment items

### Low SME Review

- general wording
- UI labels
- basic interaction structure
- generic reflection prompts

## QA Rules for This Pilot

For every course, check:

- objectives are measurable
- objectives align with assessment items
- scenarios include respectful and inclusive language
- recurring frameworks are consistent
- accessibility decision path is represented accurately
- plain language is used
- acronyms are explained
- images require alt text
- videos require captions/transcripts
- activities are keyboard-accessible where applicable
- Indigenous/cultural safety content is flagged for careful review
- final MAP activity exists

## Demo Story

The product demo should say:

> We uploaded a large accessibility training source document. Course Factory OS automatically split it into 17 course records, extracted repeated instructional structures, generated standardized course blueprints and storyboards, flagged SME review needs, and created QA and handoff checklists.
