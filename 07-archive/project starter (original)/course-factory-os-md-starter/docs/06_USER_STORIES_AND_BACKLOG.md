# 06 User Stories and Backlog

## Epic 1: Project Intake

### Story 1.1

As an instructional designer, I want to create a new course project so that I can track a development request from intake to handoff.

Acceptance criteria:

- User can create a project.
- User can add title, audience, owner, due date, output type, and risk level.
- Project appears on dashboard.

### Story 1.2

As an instructional designer, I want the system to generate a lightweight analysis summary so that I can document the project rationale even when formal analysis is skipped.

Acceptance criteria:

- User can enter request notes.
- AI generates audience, need, risks, success criteria, and recommended ID model.
- User can edit and save the output.

## Epic 2: Source Material Management

### Story 2.1

As an instructional designer, I want to upload a source document so that the system can extract useful course content.

Acceptance criteria:

- User can upload a file or paste text.
- File appears in Source Material Hub.
- Parse status is visible.

### Story 2.2

As an instructional designer, I want the system to split a large source document into guide/course records so that I can manage a course series efficiently.

Acceptance criteria:

- AI identifies guide titles and numbers.
- AI creates course guide records.
- User can review and edit guide records.

## Epic 3: Blueprint Generation

### Story 3.1

As an instructional designer, I want AI to generate a course blueprint so that I can quickly define the course architecture.

Acceptance criteria:

- Blueprint includes course goal, learning objectives, module structure, assessment strategy, interaction strategy, and seat time.
- Output is editable.
- Blueprint can be approved.

## Epic 4: Storyboard Builder

### Story 4.1

As an instructional designer, I want AI to generate a storyboard from a blueprint so that I can start development from a structured draft.

Acceptance criteria:

- Storyboard screens are generated in table format.
- Each screen includes on-screen text, narration, interaction, visual direction, assessment/reflection, accessibility note, SME review level, and status.
- User can edit each field.

### Story 4.2

As an instructional designer, I want to link storyboard screens to source sections so that SMEs can verify where content came from.

Acceptance criteria:

- Storyboard screen can store source section IDs.
- Source excerpt can be displayed in screen detail view.

## Epic 5: SME Review

### Story 5.1

As an instructional designer, I want AI to generate targeted SME review questions so that SMEs focus on the content that actually needs their expertise.

Acceptance criteria:

- Questions are generated by screen or section.
- Questions are tagged by risk level.
- User can edit and approve review questions.

### Story 5.2

As an instructional designer, I want high-risk content flagged for SME review so that clinical, policy, compliance, and sensitive content is not missed.

Acceptance criteria:

- Storyboard screens have SME review levels.
- High and critical items appear in a review queue.

## Epic 6: Assessment and Evaluation

### Story 6.1

As an instructional designer, I want the system to generate assessments aligned with learning objectives so that the course measures intended outcomes.

Acceptance criteria:

- AI generates formative and summative questions.
- Each question links to a learning objective.
- Each question includes feedback.

### Story 6.2

As an instructional designer, I want activities mapped to Kirkpatrick levels so that evaluation planning is built into the design process.

Acceptance criteria:

- Activity or assessment items can have Kirkpatrick level.
- System suggests Level 1-4 evaluation tools.

## Epic 7: QA

### Story 7.1

As an instructional designer, I want automated accessibility and ID QA checks so that issues are found before development is complete.

Acceptance criteria:

- QA checks generate issues.
- Issues have category, severity, and recommendation.
- User can mark issues resolved.

## Epic 8: Export

### Story 8.1

As an instructional designer, I want to export a Word storyboard so that I can share a familiar review document with stakeholders.

Acceptance criteria:

- Export includes course info, objectives, storyboard table, SME questions, and QA notes.

### Story 8.2

As an instructional designer, I want to export an Excel tracker so that I can manage production status and review progress.

Acceptance criteria:

- Export includes all storyboard screens and statuses.
- Export includes SME review and QA fields.
